# 地图病害数据展示简易方案

## 1. 问题说明

地图页面需要展示两类数据：

- 路网数据
- 路网上的病害数据

但是病害数据量较大，如果一次性全部查询并展示，会导致：

- 接口响应慢
- 页面卡顿
- 浏览器内存占用过高
- 严重时页面卡死

因此，病害数据不能一次性全部加载。

---

## 2. 核心思路

核心原则：

```text
地图看到哪里，就查询哪里；
地图缩放到合适级别，才展示详细病害。
```

也就是说：

- 不查询全部病害
- 只查询当前地图视野范围内的病害
- 缩放级别较小时展示统计或聚合
- 缩放级别较大时展示具体病害

---

## 3. 前端处理方案

### 3.1 获取地图当前范围

用户拖动或缩放地图结束后，前端获取当前地图可视范围：

```text
minLng：最小经度
minLat：最小纬度
maxLng：最大经度
maxLat：最大纬度
zoom：当前地图缩放级别
```

然后请求后端接口：

```http
GET /api/map/diseases?minLng=113.1&minLat=22.1&maxLng=114.1&maxLat=23.1&zoom=14
```

---

### 3.2 请求防抖

地图拖动过程中不要一直请求接口。

建议在地图停止移动后再请求，例如延迟 300ms：

```javascript
let timer = null

function onMapChange() {
  clearTimeout(timer)

  timer = setTimeout(() => {
    loadDiseaseData()
  }, 300)
}
```

---

## 4. 后端处理方案

### 4.1 按地图范围查询

后端根据前端传入的地图范围，只查询当前范围内的病害数据。

不要这样做：

```text
查询全部病害数据
```

应该这样做：

```text
只查询当前地图窗口内的病害数据
```

---

### 4.2 增加空间索引

如果病害数据存储在 MongoDB 中，建议使用 GeoJSON 格式保存坐标。

病害数据示例：

```json
{
  "id": "D001",
  "type": "裂缝",
  "severity": "严重",
  "location": {
    "type": "Point",
    "coordinates": [113.25123, 23.18123]
  }
}
```

创建空间索引：

```javascript
db.disease.createIndex({ location: "2dsphere" })
```

---

### 4.3 控制返回数量

接口需要限制最大返回数量。

例如最多返回：

```text
1000 条
```

如果当前区域数据过多，可以返回提示：

```json
{
  "mode": "too_many",
  "message": "当前区域数据太多，请放大地图查看",
  "data": []
}
```

---

## 5. 按缩放级别展示

可以根据地图 zoom 级别控制展示方式：

| zoom 范围 | 展示方式 |
|---|---|
| zoom < 12 | 不展示明细，只展示数量 |
| 12 <= zoom < 15 | 展示聚合点 |
| zoom >= 15 | 展示具体病害 |

---

## 6. 接口返回示例

### 6.1 聚合数据

```json
{
  "mode": "cluster",
  "data": [
    {
      "lng": 113.25,
      "lat": 23.18,
      "count": 56
    }
  ]
}
```

---

### 6.2 具体病害数据

```json
{
  "mode": "detail",
  "data": [
    {
      "id": "D001",
      "type": "裂缝",
      "severity": "严重",
      "lng": 113.25123,
      "lat": 23.18123
    }
  ]
}
```

---

## 7. 整体流程

```text
用户打开地图
    ↓
加载路网数据
    ↓
获取当前地图范围
    ↓
请求当前范围内的病害数据
    ↓
后端根据 zoom 判断返回聚合数据还是明细数据
    ↓
前端根据返回结果渲染地图
    ↓
用户拖动或缩放地图后重新请求
```

---

## 8. 总结

本方案的重点是：

- 不一次性查询全部病害
- 只查询当前地图范围内的数据
- 根据缩放级别控制返回内容
- 限制接口最大返回数量
- 前端增加请求防抖

最终采用：

```text
地图范围查询 + 缩放级别控制 + 最大返回数量限制 + 前端防抖
```

这样可以避免页面卡死，并且方便后续扩展。
