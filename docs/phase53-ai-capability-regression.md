# Phase 53 AI 能力回归用例集

## 目标

AI 治理台已有三类偏静态能力：策略样例校验能力命中，Plan 模拟器解释一次请求的计划工具，方案质量评测检查已生成草稿质量。能力回归用例集补齐“实跑链路”：

- 从用户动作出发，真实调用 `/api/agent/map-agent/run`。
- 校验 `answerMeta`、能力 ID、必需工具、禁止工具和方案模板。
- 把一张图分析、知识问答、方案生成放进同一套可重复回归入口。

## 默认用例

- 知识问答：解释 PCI 指标，只允许知识检索，不应查询区域、病害、评定等业务工具。
- 分析路线、路段、病害、评定结果、框选区域。
- 生成路线养护报告、路段养护计划、病害处置建议、病害复核意见、评定养护建议、区域养护建议。

## 与现有治理能力的边界

- 策略样例：不执行工具，只验证治理配置中的能力命中和工具计划。
- Plan 模拟器：面向单次排障，解释某个请求为什么命中某能力和工具。
- 能力回归：执行完整 map-agent 链路，验证真实工具、模板、answerMeta 和降级状态。
- 方案质量评测：面向已保存或 fixture 草稿，检查业务证据、地图绑定和模板质量。

## 阻断规则

- 缺少 `answerMeta`、回答为空、能力 ID 不匹配。
- 必需工具未调用，或禁止工具被调用。
- 生成类能力未命中标准模板。
- `actionResult.status` 非 `SUCCESS`。
- 严格大模型模式下，LLM 未成功生成或发生降级。

## 后续增强

- 用例参数可视化编辑和环境分组。
- 回归结果保存为治理发布前置门禁。
- 将失败项一键跳转到 trace 执行过程抽屉。
