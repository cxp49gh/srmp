# 智路养护平台（SRMP）项目概述

**系统名称：** 智路养护平台
**英文名：** SmartRoad Maintenance Platform
**简称：** SRMP
**版本：** V1.0

---

## 1. 项目定位

智路养护平台一期聚焦于 **GIS 一张图 + 数据接入 + 道路资产展示 + 病害分析 + 巡检评定 + AI 大模型辅助分析**，旨在为公路养护管理人员提供一个可接入数据、可展示路网、可分析病害、可辅助决策的智能化平台。

核心数据链路：

```
数据导入 → 道路资产入库 → GIS一张图展示 → 病害/评定结果上图 → 统计分析 → 大模型智能分析
```

## 5. 核心业务对象

### 5.1 道路资产（三级结构）

```
路线 (road_route)
  └── 路段 (road_section)
        └── 评定单元 (road_evaluation_unit)
```

### 5.2 巡检与病害

```
巡检任务 (inspection_task)
  ├── 巡检轨迹 (inspection_track)
  ├── 病害记录 (disease_record)
  └── 评定结果 (assessment_result)
        └── 指标结果 (index_result)
```

### 5.3 评定指标体系

| 指标 | 名称 |
|------|------|
| MQI | 公路技术状况指数 |
| SCI | 路基技术状况指数 |
| PQI | 路面技术状况指数 |
| BCI | 桥隧构造物技术状况指数 |
| TCI | 沿线设施技术状况指数 |
| PCI | 路面损坏状况指数 |
| RQI | 路面行驶质量指数 |
| RDI | 路面车辙深度指数 |
| PBI | 路面跳车指数 |
| PWI | 路面磨耗指数 |
| SRI | 路面抗滑性能指数 |
| PSSI | 路面结构强度指数 |

## 6. 数据库设计

### 6.1 核心表清单

| 类别 | 表名 | 说明 |
|------|------|------|
| 基础 | tenant / sys_user / sys_org / sys_dict_type / sys_dict_item | 租户、用户、机构、字典 |
| 道路资产 | road_route / road_section / road_evaluation_unit | 路线、路段、评定单元 |
| 巡检 | inspection_task / inspection_track | 巡检任务与轨迹 |
| 病害 | disease_type_dict / disease_record | 病害类型与记录 |
| 评定 | index_result / assessment_result / road_condition_statistics | 指标、综合评定、路况统计 |
| 导入 | data_import_task / data_import_field_mapping / data_import_error_log | 导入任务、字段映射、错误日志 |
| 文件 | file_resource / file_relation | 文件与业务关联 |
| 大模型 | agent_session / agent_message / agent_tool_call_log / knowledge_document / knowledge_chunk | 会话、消息、工具调用、知识库 |

### 6.2 多租户设计

所有业务表含 `tenant_id` 字段，核心唯一索引包含租户 ID，查询自动按租户隔离。

### 6.3 空间字段

使用 PostGIS 存储空间几何类型（LineString、Point、Polygon），通过 GIST 索引加速空间查询。

## 9. AI 大模型分析

### 9.1 核心能力

- 路况问答（"G210 今年路况怎么样？"）
- 病害热点分析（"哪些路段坑槽最多？"）
- 评定结果解读（解释 MQI、PQI、PCI 变化原因）
- 养护建议生成
- 自动报告生成
- 地图联动（查询结果高亮到地图）

### 9.2 技术路径

RAG + 工具调用 + 受控 Text2SQL，通过 AgentTool 封装业务查询能力。

### 9.3 安全边界

```
不直接修改业务数据
不自动生成正式养护计划
不自动派发工单
所有建议仅作为辅助决策
```

## 12. 项目一期建设总结

一期系统的核心不是工单流程，而是先把数据和地图能力建设起来。最重要的四个能力是：

```
1. 数据能导入
2. 道路资产能上图
3. 病害和评定结果能分析展示
4. 大模型能基于真实数据做问答和辅助分析
```

最终形成：

> 一个以 GIS 一张图为核心入口，
> 以道路资产、病害、巡检评定结果为核心数据，
> 以大模型智能分析为辅助能力，
> 可支撑养护管理人员进行路况研判和辅助决策的智能化平台。
