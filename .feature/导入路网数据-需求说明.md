## 3. 文件格式规格

### 3.1 TAR

- **【已定】** 仅支持 **非压缩** `.tar`（不支持 `.tar.gz` 除非实现时顺手加 `TarArchiveInputStream` 检测 gzip magic，本期不要求）。
- tar 内允许子目录；递归查找 **`.shp` 文件**。
- **【已定】** 若找到 **多于一个** 不同基名的 shp（例如 `a.shp` 与 `b.shp` 且不是同一组 sidecar），返回错误：`「压缩包内存在多组 Shapefile，请只保留一组路网数据」`。
- **【已定】** 「一组」判定：同一目录下存在 `{base}.shp` + `{base}.dbf` + `{base}.shx`。**缺少 dbf/shx 则失败**。
- **【已定】** 必须存在 `{base}.prj`；**若缺失则失败**（错误文案：`「缺少 .prj 文件，无法确定坐标系」`），避免默认错坐标。

### 3.2 DBF 属性列（大小写不敏感）

逻辑列名归一化：**trim + upper** 后与下表匹配。

| 逻辑列名 | 必填 | 说明 |
|----------|------|------|
| `ROUTE_NO` | 是 | 路线编号 → `route_code` |
| `NAME_CHN` | 否* | 中文名 |
| `NAME_ENG` | 否* | 英文名 |
| `LENGTH` | 否 | 长度，**【已定】** 单位为 **米 (m)** → `length_km = LENGTH / 1000`；若为空则用几何长度（4326 下 `ST_Length` 地理或平面近似由实现二选一，**优先用属性 LENGTH**） |
| `START_MP` | 否 | → `start_stake` |
| `END_MP` | 否 | → `end_stake` |
| `ID` | 否 | 写入 `remark` 前缀 `sourceId=...`（若超长则截断 remark） |
| `DIRECTION` | 否 | 拼入 `remark` |
| `IG_DIR` | 否 | 拼入 `remark` |

\* **【已定】** `route_name`：`NAME_CHN` 非空则用之；否则 `NAME_ENG`；否则 **失败**（该行错误：`「要素缺少 NAME_CHN 与 NAME_ENG」`），**不用** `ROUTE_NO` 占位（避免与真实名称混淆）。

### 3.3 几何类型 **【已定】**

| SHP 几何 | 处理 |
|-----------|------|
| `LineString` | 转 WKT → 转 4326 → 入库 |
| `MultiLineString` | 取 **长度最长** 的一条 `LineString` 作为该要素几何；若子线为空则失败 |
| `Point` / `Polygon` 等 | **跳过该要素**，计入 `skippedCount`，不失败整包；若 **全部** 被跳过则整包失败 |

## 4. 字段映射（SHP 属性 + 几何 → `RoadRouteSaveDTO` / 实体）

生成 `RoadRouteSaveDTO` 时：

| DTO 字段 | 来源 | 规则 |
|----------|------|------|
| `routeCode` | `ROUTE_NO` | `trim()`；全角字母数字可选转半角 |
| `routeName` | `NAME_CHN` / `NAME_ENG` | 见 §3.2 |
| `routeType` | 无 | **【已定】** 固定字面量 **`OTHER`**（满足 `@NotBlank`）。**同一次 PR** 在 `init_dict.sql` 追加：  
`INSERT INTO sys_dict_item(tenant_id, dict_type, item_code, item_name, sort_no) VALUES ('default','route_type','OTHER','外部路网导入',99) ON CONFLICT (tenant_id, dict_type, item_code) DO NOTHING;`  
（列集合以 `schema.sql` 中 `sys_dict_item` 为准，若 INSERT 需带 `id` 则显式 `gen_random_uuid()` 或与现有脚本风格一致。） |
| `adminGrade` | — | `null` |
| `technicalGrade` | — | `null` |
| `startStake` | `START_MP` | `BigDecimal`，空则 `null` |
| `endStake` | `END_MP` | 空则 `null`；若 `startStake` 与 `endStake` 均非空且 `startStake > endStake`，**交换**并记一条 warning 字符串 |
| `lengthKm` | `LENGTH` | 米 → 千米；空则尝试用几何折线长在目标 CRS 下估算（实现自选公式，写入前保留 3 位小数） |
| `adcode` / `manageOrgId` | — | `null` |
| `geomWkt` | shp 几何 | CRS 由 `.prj` 解析 → **必须变换到 EPSG:4326** → `LINESTRING(lon lat, ...)` WKT |
| `remark` | 多字段拼接 | 格式 **【已定】**：`parts = []`；若有 `ID` 则 `parts.add("sourceId=" + ID)`；若有 `DIRECTION` 则 `parts.add("direction=" + DIRECTION)`；若有 `IG_DIR` 则 `parts.add("igDir=" + IG_DIR)`；`String.join(" | ", parts)`；总长超过 **2000 字符** 则截断至 2000 |

**数据库写入路径**：与现有一致 —— 调用 `RoadRouteService.create(dto)` 或 `RoadRouteService.update(id, dto)`；不要绕过 Mapper 手写 SQL 插入几何（除非与 `insertWithGeom` 行为一致）。

## 5. 算法步骤（伪代码级别）

```
POST import-network(file):
  assert file.originalFilename.toLowerCase().endsWith(".tar")
  tempDir = Files.createTempDirectory("srmp-network-import-")
  try:
    safeUntar(file.inputStream, tempDir)   // 校验每个 entry 规范化路径不包含 ".."
    shpBase = findSingleShapefileGroup(tempDir)  // 唯一 {base}
    if shpBase == null: throw BizException("...")

    features = readFeaturesWithGeoTools(shpBase)  // dbf + shp + prj
    validateDbfHasColumn(features.schema, "ROUTE_NO")

    rows = []
    seenCodes = new HashSet()
    for each f in features:
      attrs = normalizeKeys(f.properties)
      routeCode = normalizeRouteCode(attrs["ROUTE_NO"])
      if routeCode is blank: throw row error OR collect — 【已定】collect row error, end loop then if any hard error fail
      if seenCodes.contains(routeCode): throw BizException("压缩包内路线编号重复: " + routeCode)
      seenCodes.add(routeCode)
      dto = buildDto(attrs, f.geometry)
      rows.add(dto)

    // 二次：桩号、名称已在 buildDto 校验
    @Transactional
    for dto in rows:
      existingId = roadRouteMapper.selectIdByTenantAndRouteCode(tenantId, dto.routeCode)
      if existingId == null: roadRouteService.create(dto)
      else: roadRouteService.update(existingId, dto)

    return ImportNetworkResultVO(...)
  finally:
    deleteRecursively(tempDir)
```

**注意**：`selectIdByTenantAndRouteCode` **当前可能不存在**，需新增 Mapper 方法与 XML：`SELECT id FROM road_route WHERE tenant_id=? AND route_code=? AND deleted=false LIMIT 1`。

## 6. `route_code` 规范化 **【已定】**

```text
输入 s = ROUTE_NO.trim()
若 s 为空 → 要素级错误
否则返回 s（不做强行大写，避免改变客户编号语义）；若需与库内已有大小写冲突处理，使用 SQL `route_code` **区分大小写**匹配；若 PostgreSQL 实际为不区分大小写排序规则，以现有表唯一约束行为为准）
```

（实现注释：表唯一约束为 `(tenant_id, route_code)`，若 DB 为 citext 则注意；schema 为 `VARCHAR`，默认区分大小写。）

## 7. HTTP API 契约

### 7.1 请求

```http
POST /api/road-routes/import-network
Content-Type: multipart/form-data
X-Tenant-Id: <tenant>   # 与现网一致；若 Security 要求 Token，沿用项目全局配置
```

| Part 名 | 类型 | 必填 |
|---------|------|------|
| `file` | binary | 是 |

**Spring 配置**：若默认 `spring.servlet.multipart.max-file-size` 过小，在 `application-dev.yml` / `application.yml` 增加例如 `100MB`（与产品 §8 一致）。

### 7.2 响应体 `data` 结构（建议新建 VO）

```ts
// 逻辑类型，供前端 TypeScript 对齐
interface ImportNetworkResultVO {
  insertedCount: number
  updatedCount: number
  skippedCount: number   // 非 LineString/MultiLineString 的要素
  warnings: string[]     // 如桩号交换、几何与 LENGTH 偏差>5% 等
}
```

失败时：`R` 的 `code != 0`，`message` 为简短原因；可选 `data` 为 `{ details: string[] }`（最多 50 条避免爆体）。

在 **`srmp-road-asset`** 模块内的 **`RoadRouteController`** 增加方法即可（保持 `/api/road-routes` 前缀统一；勿在其它模块另起一套 `/api/road-routes/*`）。

## 13. 自动化测试场景（实现后自测）

1. 最小合法 tar：1 条 LineString，`ROUTE_NO=G999`、`NAME_CHN=测试` → `insertedCount=1`。  
2. 再次导入同包同编号 → `updatedCount=1, insertedCount=0`，`id` 不变（手动查 DB）。  
3. tar 内含两个不同 shp 基名 → 整包失败。  
4. 无 `.prj` → 失败。  
5. `MultiLineString` 两条线 → 取最长线成功入库。  
6. 仅 Polygon 要素 → 失败（skipped 后无有效行）。
