# 导入路段数据 — AI 实现规格书（Shapefile/TAR → road_section / road_evaluation_unit / assessment_result）

> **读者模型**：本文件供大模型或工程师直接据此生成/修改代码，不要求再向人类澄清业务意图。  
> **硬性规则**：凡标注 **【已定】** 的条目按本文实现，勿自行改语义；标注 **【可选】** 的可按注释取舍。  
> **关联计划**：`srmp/.plan/导入路段数据.plan.md`  
> **本期范围**：**【已定】** 仅实现 **路线级**、**台账级** 两类 Shapefile 的解析与入库；**公里级、百米级** 压缩包内可存在但 **必须忽略**（不解析、不报错），并在成功响应的摘要中提示「已忽略 N 组未支持级别」**【可选】**。

## 0. 执行摘要（给模型的 TL;DR）

1. **后端主归属**：**【已定】** 解析、解压、GeoTools 读 shp、租户上下文、与 `road_route` / `road_section` / `road_evaluation_unit` 的写库逻辑，放在 **`srmp-road-asset`**（包根 `com.smartroad.srmp.roadasset`）。  
2. **评定结果表**：`assessment_result` 归属 **`srmp-assessment`** 模块与表结构；**【已定】** 实现时应在 **`srmp-road-asset` 的 `pom.xml` 增加对 `srmp-assessment` 的可选/直接依赖**，通过 **`AssessmentResultService`（或等价 Service）** 写入 `assessment_result`，避免在 `road-asset` 内手写跨模块 SQL 导致边界混乱。**不要**在 `srmp-import` 的 CSV/Excel 管道中塞入本功能。  
3. **几何**：与路网导入一致，入库几何为 **EPSG:4326** 的 **2D** `LineString`（路段、单元）；`road_evaluation_unit.center_point` **【已定】** 由 `geom` 求中点或 `ST_LineInterpolatePoint(0.5)` 等价逻辑生成；若 shp 无几何列则允许为空 **【可选】**（与业务确认后再定默认失败或允许）。  
4. **前端**：在 **`srmp-web-ui`** 的 GIS 一张图（`OneMap.vue` 工具栏，与「导入路网」并列）增加「导入路段」→ 仅 **`.tar`** → **`longRequest`** 上传 → 成功刷新 **路段图层、评定单元图层**（复用现有 `loadLayerSafely` 或项目内等价 API）。  
5. **事务**：**【已定】** 单次上传 **整包原子**：任一路线级/台账级要素校验失败、或任一行写库失败 → **全部回滚**（含已插入的 `assessment_result`）。  
6. **本文档**：仅规格说明；**不**在本任务中直接生成业务代码（与计划「只生成内容」一致）。

## 2. 【已定】产品行为（PB）

| ID | 行为 |
|----|------|
| PB-1 | 前端「导入路段」仅接受 **`.tar`**（`accept=".tar"`）。 |
| PB-2 | 上传中 loading；失败展示服务端 `message`；若有结构化 `details`（行号、要素 id、字段名），前端截取展示前若干条。 |
| PB-3 | 成功：`ElMessage.success` + **刷新路段与评定单元相关图层**（及与当前查询条件一致的路由筛选）。 |
| PB-4 | **整包事务**：见 §0。 |
| PB-5 | **路线级** tar 内 **【已定】** 不允许重复 **规范化后的** `section_code`（见 §6）；重复则整包失败并列出键。 |
| PB-6 | **台账级** 同理：**【已定】** 不允许重复规范化后的 `unit_code`。 |
| PB-7 | **路线必须先存在**：台账级某行的 `route_code` 在 `road_route` 中不存在 → 整包失败。 |
| PB-8 | **【已定】** 若台账级需要挂接 `section_id`：优先在 **本包已导入的路线级结果** 中按 `(route_code, direction, start_stake, end_stake)` 精确匹配路段；若无匹配 **【已定】** 再查库中已有 `road_section`（同租户、未删除）；仍无则 **`section_id` 置空**（库表允许）并在 `AssessmentResultSaveDTO` 中同步不填 `section_id`，**不**因此失败。 |
| PB-9 | **检测年度** `year` 为 `assessment_result.year` **必填**；缺失或无法解析为整数 → 该要素失败 → 整包失败。 |

## 3. 文件格式规格

### 3.1 TAR 包结构 **【已定】**

- 与路网导入一致：**非压缩** `.tar`；递归查找所有 `.shp`。  
- **一组 Shapefile** = 同目录 `{base}.shp` + `{base}.dbf` + `{base}.shx` + **`{base}.prj` 必填**。  
- 包内允许同时存在 **多组** shp，每组通过 **基名或路径** 判定级别（§3.3）。  
- **【已定】** 若某一「本期应处理」级别的组缺少 dbf/shx/prj，整包失败。  
- **公里级、百米级**：若文件名/路径匹配 §3.3 的忽略规则，**跳过**；若无法识别且既不是路线级也不是台账级关键字，**【已定】** 视为错误：`「存在无法识别的 Shapefile 级别，请按命名约定放置路线级/台账级数据」`（避免静默丢数据）。

### 3.2 DBF 字符集 **【已定】**

- 与路网导入对齐：使用可配置项 **`srmp.import.section.dbf-charset`**（UTF-8 默认）；**【可选】** 与 `network` 共用同一 key 由产品决定，规格上推荐独立 key 以免两处编码不一致。  
- 读属性：**trim**；列名支持 **中文列名** 与 **英文逻辑名** 别名（§4 表）。

### 3.3 级别识别（基名 `base` 或相对路径，不区分大小写）**【已定】**

| 级别 | 匹配规则（满足任一即归入该类） |
|------|--------------------------------|
| 路线级 | 路径或基名包含：`路线级`、`road_section`、`section`、`路段`（与「路线级」二选一由实现择一文档化即可，避免过宽误匹配）。 |
| 台账级 | 包含：`台账级`、`evaluation_unit`、`eval_unit`、`ledger`、`评定单元`。 |
| 忽略（本期） | 包含：`公里`、`百米`、`km_unit`、`hm_unit` 等 **【可选】** 扩展词表。 |

**处理顺序**：**【已定】** 先完整处理 **路线级** 全要素（内存中建立桩号+方向 → `section_id` 映射），再处理 **台账级**。

## 5. 检测方向与字典 **【已定】**

- 读入后 **trim**；若为空则 **`BOTH`**（与表默认值一致）。  
- 若值为上行/下行等中文，**【已定】** 通过 **`init_dict`**（或现有字典 API）中 **路段方向** 类编码映射到 `UP`/`DOWN`/`BOTH`；映射不到则 **整包失败** 并列出非法值。  
- **【可选】** 与路网导入桩号方向交换规则是否联动：若台账与路线级起止桩互换检测，与路网导入 **保持一致** 的交换规则（见路网规格书 §桩号）。

## 6. `section_code` / `unit_code` 生成 **【已定】**

| 场景 | 规则 |
|------|------|
| DBF 存在「路段编码」列（别名 `SECTION_CODE`, `LDBM` 等） | 规范化 trim 后直接使用；超长截断至表长度上限并保证租户内唯一。 |
| 不存在 | `section_code = {route_code}-K{start_stake}-K{end_stake}-{direction}`，非法字符替换为 `_`，总长超过 100 时 **SHA-256 前 16 位** 后缀保证唯一（实现需文档化精确算法）。 |
| `unit_code` | 同理：优先「评定单元编码」列 `UNIT_CODE`, `DYBM`；否则 `{route_code}-U-K{start}-K{end}-{direction}` 或哈希后缀策略与上一致。 |

## 7. `assessment_result` 写入约定 **【已定】**

| 字段 | 路线级导入 | 台账级导入 |
|------|------------|------------|
| `object_type` | **`ROAD_SECTION`**（与 Agent/GIS 中 `ROAD_SECTION` 一致） | **`EVALUATION_UNIT`** |
| `object_id` | 新插入或更新后的 **`road_section.id`** | **`road_evaluation_unit.id`** |
| `route_id` / `route_code` / `direction` / `start_stake` / `end_stake` | 自 shp 与父实体复制 | 同左 |
| `section_id` | 与 `object_id` 相同 **【可选】** 或填自身 id（实现选一种并与 GIS 查询 JOIN 习惯对齐）；**【已定】** 至少 **`object_id` + `object_type` 可唯一定位** | 填匹配到的 `road_section.id`，无则 `null` |
| `unit_id` | `null` | 与 `object_id` 相同 |
| `task_id` | `null` | `null` |
| `year` | 必填 | 必填 |
| `standard_code` | 默认 **`JTG_5210_2018`**（与 schema 默认一致） | 同左 |
| `assessed_at` | `LocalDateTime.now()` **【可选】** 或用检测年度构造 `yyyy-12-31T23:59:59` | 同左 |

**Upsert**：同一 `(tenant_id, object_type, object_id, year, standard_code)` 已存在 → **更新**分数与 `zero_reason`/`grade`；不存在 → **插入**。删除软删 **【已定】** 本期不做「按包删除旧结果」，避免误删历史年度。

## 8. API 与响应 **【已定】**

- **方法/路径**：`POST /api/road-sections/import-package`（若与 REST 资源命名冲突，可改为 `POST /api/road-assets/import-sections`，但需在实现与前端统一一处）。  
- **Content-Type**：`multipart/form-data`，字段名 **`file`**。  
- **响应体**：沿用 `R<T>`；`data` **【已定】** 建议包含：`routeSectionCount`、`evaluationUnitCount`、`assessmentRowCount`、`ignoredShapefileGroups`（数组字符串）、`durationMs`。  
- **错误码**：与路网导入风格一致（400 参数、413 体积过大、500 服务器错误）。

## 9. 校验规则摘要 **【已定】**

- 数值列：桩号、长度、宽度、各指标 → `BigDecimal` 安全解析；非法格式 → 要素级错误信息含 **shp 基名 + 要素序号/FID**。  
- `start_stake` 与 `end_stake`：**【已定】** 允许起止颠倒 **仅当** 与项目路网桩号规则一致时自动交换并打日志；否则失败（与路网规格对齐）。  
- 几何：若存在则 CRS 必须有 `.prj`；变换到 EPSG:4326 与路网导入相同（水平 CRS + 2D）。  
- **租户**：`TenantContextHolder.getTenantId()` 非空。

## 12. 测试用例建议（实现阶段）

1. 最小 tar：仅路线级，1 条要素 → `road_section` + `assessment_result` 各 1 行，`object_type=ROAD_SECTION`。

7. 含公里级 shp → 忽略且不失败 **【可选】**。

## 13. 与《导入路网数据》规格的关系

- **依赖**：必须先完成路网导入或库内已有 `road_route`，否则路段导入按 PB-7 失败。  
- **复用**：TAR 扫描、PRJ 校验、Charset、GeoTools 版本、4326 2D 策略、整包事务模式，与路网规格 **保持一致**，便于同一维护者理解与排错。
