> **读者模型**：本文件供大模型或工程师直接据此生成/修改代码，不要求再向人类澄清业务意图。  
> **硬性规则**：凡标注 **【已定】** 的条目按本文实现，勿自行改语义；标注 **【可选】** 的可按注释取舍。  
> **关联计划**：`srmp/.plan/导入病害数据.plan.md`（其中 **对应模块：srmp-road-asset**、**对应数据库：`disease_record`** 为硬性边界）。  
> **本期范围**：**【已定】** 前端 **`.xlsx`** 选择并上传、成功后页面提示；后端在 **`srmp-road-asset`** 内实现 **流式** 读表；**表头在第 5 行**、**数据从第 6 行** 起解析并写入 **`disease_record`**。**【已定】** 本文档仅规格说明，**不**在本任务中直接生成业务代码。

## 0. 执行摘要（TL;DR）

1. **后端主归属**：**【已定】** **`srmp-road-asset`**（与「导入路网 / 导入路段」同一模块策略：资产侧统一承接 **文件类导入**）。在包内新增 **multipart** 接口（建议独立 Controller，见 §1.1），完成 XLSX 解析与入库调用。  
2. **持久化目标**：**【已定】** 数据库表 **`disease_record`**；实体与 DTO 以 **`srmp-disease`** 模块中的 **`DiseaseRecord` / `DiseaseSaveDTO` / `DiseaseRecordService`** 为准，**【已定】** `srmp-road-asset` 的 `pom.xml` **须增加对 `srmp-disease` 的依赖**，通过 **Service 层** 写入，**禁止**在 `road-asset` 内手写跨模块 SQL 直插 `disease_record`（与路段导入依赖 `srmp-assessment` 的边界一致）。  
3. **前端**：**【已定】** **`srmp-web-ui`** 增加 **「导入病害数据」** → 隐藏 file input、`accept` 以 **`.xlsx`** 为主 → 选文件后立即上传 → **`ElMessage.success`** 须包含 **「上传成功」** 语义（见 §2）。loading / 错误展示与现有导入按钮对齐。  
4. **Excel**：**【已定】** 表头列名与顺序见 §3；**表头 = 第 5 行**，**数据起始行 = 第 6 行**（第 1～4 行忽略）。  
5. **大文件**：**【已定】** 采用 **流式读取**（§7），避免整簿 DOM 常驻堆。  
6. **multipart 上限**：**【已定】** 依赖 **`srmp-admin`** 已配置的 Tomcat / Spring multipart 上限（与路段包等大上传一致）。

## 3. Excel 格式 **【已定】**

### 3.1 文件类型

- **【已定】** 扩展名 **`.xlsx`**；若用户选择其他类型，前端拦截或后端返回明确错误。

### 3.2 表头（第 5 行）

**【已定】** 列名（trim 后匹配）及 **从左到右顺序** 如下（与计划「文件头」一致，列间为制表符或 Excel 列顺序等价）：

| 列序 | 表头文字 | 说明 |
|------|----------|------|
| 1 | 序号 | 人工辅助列，**不参与业务主键**；可空。 |
| 2 | 路线编码 | **必填** → `route_code`。 |
| 3 | 方向 | 规范化至库内 `direction` 取值（与路线资产一致）。 |
| 4 | 桩号 | 解析规则见 §4.2。 |
| 5 | 路面材质 | 表无独立列 → **【已定】** 写入 `remark` 前缀 `路面材质:xxx;`（空则省略）。 |
| 6 | 经度 | 与纬度成对，见 §4.1。 |
| 7 | 纬度 | 同上。 |
| 8 | 高度 | 见 §5.3。 |
| 9 | 病害名称 | **必填** → `disease_name`；参与 §5.1 分类推导。 |
| 10 | 长度(m) | → `damage_length`。 |
| 11 | 宽度(m) | → `damage_width`。 |
| 12 | 面积(㎡) | → `damage_area`。 |
| 13 | 备注 | 追加至 `remark`（与路面材质等 **分号分隔**）。 |
| 14 | 图片地址 | 表无图片列 → **【已定】** 追加至 `remark`，格式 `图片:URL`（多图 **【可选】** 逗号分隔）。 |

### 3.3 数据起始行

- **【已定】** **第 6 行** 为 **第一条数据行**。  
- **第 1～4 行**：任意标题、说明、空行，**解析器跳过**。  
- **【已定】** 表头必须为 **第 5 行**；若第 5 行与 §3.2 不一致 → **整文件失败**并提示「表头不匹配，请使用标准模板」。

### 3.4 空行与结束

- **【已定】** 数据区内 **整行全空** 可跳过，不记错误。  
- 读到 sheet 物理末行即结束。

## 4. 字段映射与解析规则 **【已定】**

### 4.1 几何 `geom`

- `经度`、`纬度` 均可解析为有限小数 → 生成 **`POINT(lon lat)`**，**SRID 4326**，经 `DiseaseSaveDTO.geomWkt` 写入（与全仓 GIS 几何约定一致）。  
- **任一缺失或非法** → **该行失败**（不允许仅用桩号伪造点位）。  
- **高度**：见 §5.3。

### 4.2 桩号 → `start_stake` / `end_stake`

- **单值**（无区间符号）→ `start_stake = end_stake =` 解析值。  
- **区间**（含 `～` `~` `—` `-` 等 **【已定】** 实现需列明支持字符集）→ 拆成起、止；解析失败 → **行失败**。  
- 桩号格式支持与路网/路段模块 **【可选】** 复用同一工具类；若无则在本服务内实现最小解析（数字 + K 桩 **【可选】**）。

### 4.3 `route_id` 与 `route_code`

- 按 **租户** 查询 **`road_route`**（查询在 **road-asset** 已有 `RoadRouteMapper` / `RouteCodeNetworkLookup`，**【已定】** 不依赖 disease 模块查路线）。
- **【已定】** 若 Excel 中路线编码在末尾附带 **上下行标识**（常见 **`AB`**，或单侧 **`A`/`B`**，大小写不敏感），而路网 **`route_code` 不含该后缀**：查库时 **先按原文**，再按 **`RouteCodeNetworkLookup`** 展开候选（剥后缀等，与路段导入一致）。
- **命中路网**：**`route_id`** = `road_route.id`；**`disease_record.route_code`** = 库内 **`road_route.route_code`**（规范编码，与路网一致）。
- **未命中**：**`route_id`** = `NULL`；**`disease_record.route_code`** = 按 **剥尽上下行后缀** 后的导入编码（`stripAllDirectionSuffixes`），与历史「与路网一致」表述兼容；**`direction`**：仍优先由编码末尾后缀推断（`A`→`UP`、`B`→`DOWN`、`AB`→`BOTH`），**无后缀**时由 Excel **「方向」** 列解析（§3.2）。
- **不存在**：**【已定】** 仍写入该行（`route_id` 空）；服务端 **WARN** 汇总；接口返回 **`skippedMissingRouteCount`** 等（与实现 VO 一致）；**不**因此终止整文件。*（注：早期规格若写「跳过该行」，以当前实现「仍入库 + 计数」为准。）*

## 5. `disease_record` 其余字段 **【已定】**

### 5.1 `disease_category` / `disease_type`（表 NOT NULL）

Excel 仅 **病害名称**，**【已定】** 规则：

1. 在租户下查询 **`disease_type_dict`**（通过 **`srmp-disease`** 已有字典服务/Mapper，**【已定】** 不在 road-asset 手写 `disease_type_dict` SQL）：按 `disease_name` / `disease_code` **精确 → 模糊唯一** 匹配。  
2. **命中** → 填字典给出的 `disease_category`、`disease_type`（字段名与实体一致）。  
3. **未命中** → `disease_category = '未分类'`，`disease_type = 'EXCEL_IMPORT'`（或统一常量，实现时固化在一处）。

### 5.2 `severity` / `quantity` / `measure_unit`

- `severity`：Excel 无 → **空**。  
- `quantity`：**【已定】** 优先 **面积(㎡)**；面积空则用 **长度(m)**；皆空则 `quantity` 空。  
- `measure_unit`：与数量语义一致时填 **`㎡`** 或 **`m`**；`quantity` 空则 `measure_unit` 空。

### 5.3 `damage_depth` 与「高度」

- **【已定】** 默认将 **高度** 映射为 **`damage_depth`**（数值按 **米**）。  
- **【可选】** 若业务确认该列为 **海拔高程** 而非坑槽深度，则仅写入 `remark`：`高度(海拔)=...`，`damage_depth` 置空（需配置开关或注释约定）。

### 5.4 其他

| 字段 | 规则 |
|------|------|
| `task_id` | **【可选】** 空；或前端 query 传入后写入。 |
| `section_id` / `unit_id` | **【已定】** 本期 **空**。 |
| `lane_no` | **【已定】** 空。 |
| `source` | **【已定】** 常量 **`EXCEL_IMPORT`**。 |
| `confidence` | **【已定】** 空。 |
| `status` | **【已定】** 默认 **`UNPROCESSED`**。 |
| `verified` | **【已定】** `false`。 |

## 6. 校验与事务 **【已定】**

### 6.1 行级校验（摘录）

| 规则 | 说明 |
|------|------|
| V-1 | `路线编码` 空 → 行失败。 |
| V-2 | `病害名称` 空 → 行失败。 |
| V-3 | 经纬度非法或缺失 → 行失败。 |
| V-4 | 数值列非数 → 行失败。 |
| V-5 | `路线编码` 在 `road_route` 不存在 → **仍写入**，`route_id` 为空；计入 `skippedMissingRouteCount` / **`warnings`**（见 §4.3）。 |

### 6.2 事务边界

- **【已定】** **单次上传原子**：任一行失败 → **整批回滚**（已插入的全部撤销）。  
- **【可选】** 超大文件若拆批提交，**不在本期**。

### 6.3 错误响应

- 与项目统一 **`R`** 包装一致；`data.details` 为字符串列表，**【已定】** 每项含 **Excel 行号（1-based，含表头行号）** 与原因。

## 7. 大文件与流式读取 **【已定】**

### 7.1 禁止

- **【已定】** 禁止 **`WorkbookFactory.create(InputStream)`** 类整表 DOM 加载作为 **唯一** 路径处理大 xlsx（易导致 OOM）。

### 7.2 推荐实现（编码阶段选用）

| 方案 | 说明 |
|------|------|
| A | **Apache POI `XSSFReader` + SAX/`SheetContentsHandler`**：按行事件处理，`rowNum == 4`（0-based）校验表头即 **物理第 5 行**；`rowNum >= 5`（0-based）即 **物理第 6 行** 起转 DTO。 |
| B | **EasyExcel**：`ReadListener`，在 `invoke` 内 **第 5 行**（索引 4）校验表头；**第 6 行**（索引 5）起为数据；第 1～4 行（索引 0～3）跳过。 |

### 7.3 内存与批量

- **【已定】** 流式按行解析；内存中每积累 **5000** 条有效数据即批量落库，再读后续行，**不要求用户拆文件**；全程仍在 **同一事务** 内。**路线编码在路网不存在**时 **仍落库**（`route_id` 空），计入 `skippedMissingRouteCount`；若存在 **其它行级校验错误**（如经纬度非法），则 **整事务回滚**（含此前已写入的同请求数据）。  
- **【可选】** 若需防护超大恶意文件，可另设整文件行数上限（当前实现不设强制上限）。

## 9. 测试清单（实现后）**【已定】**

1. 合法最小文件：1 行数据 → 成功 + 「上传成功」。  
2. 表头错一列 → 整文件失败。  
3. 第 6 行首条数据、第 5 行为表头、1～4 行乱内容 → 成功。  
4. `路线编码` 不存在于 `road_route` → **仍成功**，`route_id` 为空，`skippedMissingRouteCount` / WARN 可核对；**命中**时 **`route_code` 与库内 `road_route.route_code` 一致**（§4.3）。  
5. 经纬度缺失 → 行失败 → 事务回滚 → 库中无新增。  
6. 字典命中 / 未命中各一条。  
7. **【可选】** 大文件压测（内存与耗时）。
